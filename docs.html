How AIKnow-IT Works

Technical overview of our platform architecture, API integrations, and content pipeline.

Platform Overview

AIKnow-IT is a cloud-based platform that automates the creation and distribution of kid-safe short-form video content. The platform integrates AI content generation with social media publishing APIs to provide an end-to-end content pipeline for creators and educational publishers.

Architecture

The platform consists of the following components:

  • Content Generation Engine: Uses OpenAI GPT-4 for script and storyboard generation, and OpenAI Sora for video creation.
  • Content Safety Layer: A multi-step validation system that checks all generated content against kid-safety guidelines before publishing.
  • Media Storage: Secure cloud storage (Supabase) for temporary video hosting, with automatic cleanup after publishing.
  • Publishing Engine: Handles OAuth authentication, video upload, and status tracking across TikTok, YouTube, and Instagram.
  • Notification System: Email-based alerts for publishing status, errors, and account issues.

Content Generation Pipeline

Step 1: Script Generation

The system sends a prompt to GPT-4 with strict content guidelines and a rotating theme category. The AI generates an original script including characters, scenes, captions, and hashtags. All scripts are validated for required fields and safety compliance.

Step 2: Video Generation

The validated script is converted into a Sora video generation prompt. Videos are generated in vertical 9:16 format (720×1280) at 20–25 seconds — optimized for short-form platforms. The system polls for completion with configurable timeout handling.

Step 3: Quality Check

Generated videos are stored temporarily and made available for review. The content safety layer performs a final check before the video enters the publishing pipeline.

TikTok Integration

Our TikTok integration uses the official TikTok Content Posting API with the following flow:

Authentication

Users authorize AIKnow-IT through TikTok's OAuth 2.0 authorization flow. We request the video.publish scope, which grants permission to publish video content on the user's behalf. No other scopes are requested.

Publishing Flow

  1. Initialize post: We call the Direct Post initialization endpoint with the video URL and metadata (title, description, privacy level).
  2. Upload/URL pull: TikTok fetches the video from our secure public URL.
  3. Status tracking: We poll the publish status until the post is confirmed as live or an error is reported.

Token Management

Access tokens are securely stored and automatically refreshed. If a token expires or is revoked, the user is notified and prompted to re-authorize.

YouTube Integration

Videos are uploaded via the YouTube Data API v3 using OAuth 2.0 credentials. Uploads include the video title (with #Shorts tag), description, tags, and category metadata. Videos are set to the appropriate privacy level and made-for-kids designation.

Instagram Integration

Instagram Reels are published using the Instagram Graph API in a two-step process:

  1. Create media container: Initialize the Reel upload with the video URL and caption.
  2. Publish: After processing (typically 15–30 seconds), publish the container to the user's feed.

Content Safety Framework

Every piece of content goes through multiple safety layers:

  • Prompt-level guardrails: AI generation prompts include explicit rules against copyrighted content, violence, adult themes, and real people.
  • Output validation: Generated scripts are parsed and validated for required fields and content policy compliance.
  • Human review option: Users can review and approve content before publishing.
  • Platform compliance: All content is designed to comply with the community guidelines of TikTok, YouTube, and Instagram.

Error Handling

The platform includes comprehensive error handling at every stage:

  • Script generation failures trigger a notification with error details.
  • Video generation timeouts (after ~10 minutes) trigger an alert with the Sora job status.
  • Publishing failures to individual platforms are tracked separately — a failure on one platform does not block publishing to others.
  • All errors are reported to the user via email with actionable details.

Data Flow Summary

The following data flows occur during a typical content generation and publishing cycle:

  1. User preferences and theme → sent to GPT-4 (no personal data included)
  2. GPT-4 response → validated and stored as content metadata
  3. Sora prompt → sent to OpenAI Sora (no personal data included)
  4. Generated video → stored temporarily in Supabase
  5. Video URL + metadata → sent to TikTok, YouTube, and Instagram APIs using user's OAuth tokens
  6. Publishing status → tracked and reported to user
  7. Temporary video storage → cleaned up after 90 days
